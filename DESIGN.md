# 🍓 ICHIGO QUEST — ゲーム設計書

分岐マップを探索する**ターン制ローグライクRPG**。1ラン5〜10分でサクッと遊べる。
ICHIGO(Optimism上のERC-20)は「払うだけ型」で課金に使う。

## コンセプト
- ジャンル: ターン制バトル × ローグライク(分岐マップ探索)
- バトル: パーティのキャラを操作しつつ、毎ターン**手札のスキルカード**を切る「いいとこ取り」方式
- 規模: 1ラン5〜10分。死んだらやり直し、何度も遊ぶ
- 技術: 素のHTML + JS(+ ethers.js v6 CDN)。ビルド不要・Vercel公開。データは localStorage

## 世界観(いちごテーマ)
いちご畑を荒らす「カビ・害虫」から畑を守る、お菓子の仲間たちの物語。

- 味方(ヒーロー)例:
  - 🍓 イチゴ剣士 … バランス型アタッカー
  - 🍰 ショートケーキ魔導士 … 全体攻撃の魔法
  - 🥛 ミルク僧侶 … 回復・防御支援
  - 🍫 チョコ盗賊 … 素早い多段攻撃
- 敵例: カビ、ナメクジ、いちご泥棒カラス、害虫 / ボス: カビゴーレム王

## バトル(ハイブリッド方式)
- 味方パーティ最大3人。各キャラに HP と固有スキルがある
- 各キャラのスキルが「カード」となり、共有の山札を作る
- 1ターンの流れ:
  1. 手札を引く(例: 5枚)/ エナジー「あまみ」を回復(例: 3)
  2. カードをプレイ(攻撃・スキル・回復)。各カードはあまみを消費。対象(敵/味方)を選ぶ
  3. ターン終了 → 敵が行動(次の行動は事前表示=テレグラフ)
- キャラはHP0で戦闘不能。敵を全滅で勝利。全滅で敗北(ラン終了)

## ラン構造(分岐マップ)
- 1ラン = 約3層の分岐ノードマップ(Slay the Spire 風)
- ノード種別: ⚔️戦闘 / 💎エリート(強敵) / ❓イベント / 🛒ショップ / 🛏️休憩(回復・強化) / 👑ボス(最後)
- 各ステップで進む道を選ぶ=探索の楽しさ
- 戦闘報酬: 新カード / ゲーム内ゴールド / レリック(永続強化)
- ゴールドはラン内で使用(オフチェーン)。死ぬとランの進行はリセット

## ICHIGO の使い道(払うだけ型・ガス最小)
- 🎰 **ガチャ**(既存の仕組みを流用): 新ヒーロー / レアカードを引いて手持ちを強化
- 🔓 高難度ダンジョンの解放
- ❤️ 全滅時の**復活 / コンティニュー**
- 🛒 プレミアムショップの特別アイテム
- 支払いは全て「集約用ウォレット(方法C)」へ1回送金。5:5精算は後でまとめて手動

## 永続要素(メタ進行)
